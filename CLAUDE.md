## Git 自動化ルール（このプロジェクント限定）

> **このプロジェクトでは、作業の節目ごとに Claude が自動でコミット＆プッシュを実行する。**
> ユーザーへの個別確認は不要。ただし下記の安全条件を必ず守ること。

### 自動コミット＆プッシュのタイミング

以下の条件を**すべて満たす**ときに、自動でコミット＆プッシュを行う：

| 条件 | 詳細 |
|------|------|
| ✅ 作業単位が完結している | 1つの機能追加・バグ修正・コンテンツ更新が完了した時点 |
| ✅ HTMLが構文的に壊れていない | タグの開閉が正しい・ページが表示可能な状態 |
| ✅ 変更ファイルが意図した範囲内 | 無関係なファイルを巻き込んでいない |
| ✅ 秘密情報が含まれていない | APIキー・パスワード等がコードに混入していない |

### 自動コミット＆プッシュの手順

作業完了後、以下の手順を自動で実行する：

```powershell
# ステップ1: 変更内容を確認（差分チェック）
git diff
git status

# ステップ2: 変更ファイルをステージング（関連ファイルのみ）
git add <変更したファイル>

# ステップ3: 日本語コミットメッセージでコミット
git commit -m "種別: 変更内容の要約（日本語）"

# ステップ4: main ブランチへプッシュ（本番に即時反映）
git push origin main
```

### コミットメッセージのルール

親 `CLAUDE.md` のルールに従い、日本語で記述する：

```
feat: プロフィールページに研究テーマを追加
fix: スマホ表示でメニューが崩れる問題を修正
docs: README に開発手順を追記
chore: .gitignore に一時ファイルを追加
```

### 自動化しない（手動確認が必要な）操作

以下の操作は **たとえこのプロジェクトでも** 自動実行せず、必ずユーザーに確認する：

- `git push --force`（強制プッシュ）
- `git reset --hard`（変更の完全破棄）
- ファイルの削除（`git rm` や OS上のファイル削除）
- ブランチの削除・強制マージ

### 作業完了後の報告フォーマット

コミット＆プッシュが完了したら、以下の形式でユーザーに報告する：

```
✅ コミット＆プッシュ完了
　コミットメッセージ: feat: プロフィールページに研究テーマを追加
　変更ファイル: profile/profile.html
　本番反映: https://kentaro-sk.github.io（数秒〜1分で反映）
```

## このプロジェクト固有のコーディング規約

### HTML
- インデントはスペース2文字
- 属性値は必ずダブルクォートで囲む（`class="foo"` ○ / `class='foo'` ×）
- 文字コードは `utf-8` を維持する
- 外部ライブラリは既存の CDN バージョンを維持する（勝手にアップデートしない）

### CSS
- 新しいスタイルは各ページの `stylesheet.css` に追記する
- 共通スタイルは `css/stylesheet.css` に追記する
- インラインスタイル（`style="..."` 属性）は使用しない

### JavaScript
- jQuery 3.4.1 を使用（バージョン変更禁止）
- `var` ではなく `const` / `let` を使用する
- すべての関数・処理ブロックに日本語コメントを付ける（親 CLAUDE.md のルール厳守）
